# Inhalte transformieren

Wandelt einen Ausgangstext in plattformgerechte Formate um — mit KI-gestuetzter Anpassung von Ton, Laenge und Struktur.

## Unterstuetzte Plattformen

| Plattform | Beschreibung | Besonderheiten |
| --- | --- | --- |
| **LinkedIn Post** | Professioneller Business-Post | Hook, max 3000 Zeichen, 3-5 Hashtags, CTA |
| **Twitter/X Thread** | Kompakter Thread | Nummerierte Tweets, 280 Zeichen pro Tweet, Emojis |
| **Medium Blog** | Langer Storytelling-Artikel | Persoenliche Erzaehlweise, Bildvorschlaege |
| **Reddit Post** | Community-Diskussionsbeitrag | TL;DR, Markdown, authentischer Ton |
| **dev.to Artikel** | Developer-Community-Artikel | "Was du lernst"-Abschnitt, Code-Beispiele, Tags |
| **GitHub Discussion** | Technische Diskussion | Code-Bloecke, Umgebungsdetails |
| **GitHub Blog** | Markdown-Blogpost | YAML-Frontmatter, Inhaltsverzeichnis, SEO |
| **YouTube Beschreibung** | SEO-optimierte Videobeschreibung | Timestamps, Ressourcen-Links, Tags (max 15) |
| **Blog Post** | Umfassender Blog-Artikel | YAML-Frontmatter, SEO, "Weiterführendes"-Abschnitt |

## Ablauf (4 Schritte)

### Schritt 1: Quelle eingeben

- Text direkt eingeben oder einfuegen
- Datei hochladen (Markdown, HTML, DOCX, TXT und mehr)
- Aus dem Editor uebernehmen
- Multi-Tab: Mehrere Dokumente gleichzeitig offen

**Shortcuts (Editor in Schritt 1):**

- **Cmd/Ctrl + S** — Speichern
- **Cmd/Ctrl + Shift + S** — Speichern unter
- **Cmd/Ctrl + W** — Aktiven Tab schliessen
- **Cmd/Ctrl + G** — Gliederung ein-/ausblenden
- **Esc** — Gliederung schliessen

### Schritt 2: Plattform waehlen

- Eine Plattform fuer Einzeltransformation
- Mehrere Plattformen im **Multi-Plattform-Modus** — alle gleichzeitig transformieren

### Schritt 3: Stil konfigurieren

**Tonalitaet:**

| Option | Beschreibung |
| --- | --- |
| Professional | Formell, business-gerecht |
| Casual | Locker, gespraechig |
| Technical | Praezise, detailliert |
| Enthusiastic | Energisch, motivierend |

**Laenge:**

| Option | Beschreibung |
| --- | --- |
| Short | 1-2 Absaetze |
| Medium | 3-5 Absaetze |
| Long | Voller Artikel |

**Zielgruppe:**

| Option | Beschreibung |
| --- | --- |
| Anfaenger | Einfache Sprache, Erklaerungen |
| Intermediate | Standardniveau |
| Experten | Fachsprache, weniger Kontext |

**Sprache:** Deutsch, English, Espanol, Francais, Italiano, Portugues, Chinesisch, Japanisch, Koreanisch, Russisch

### Schritt 4: Ergebnis

- Transformierter Text wird angezeigt
- Inline bearbeitbar
- Im Multi-Plattform-Modus: Ein Tab pro Plattform

**Aktionen:**

- **Kopieren** — In die Zwischenablage
- **Download** — Als Datei speichern
- **Bearbeiten** — Zurueck in den Editor
- **Posten** — Direkt veroeffentlichen (API-Key erforderlich)
- **Reset** — Von vorn beginnen

## Multi-Plattform-Modus

1. In Schritt 2 mehrere Plattformen auswaehlen
2. Alle Transformationen laufen parallel
3. Ergebnis zeigt Tabs fuer jede Plattform
4. Jeder Tab ist einzeln bearbeitbar und postbar

## Tipps fuer bessere Ergebnisse

- **Klare Eingabe**: Kurze Saetze, wichtige Fakten als Stichpunkte
- **Technische Begriffe**: Explizit angeben, damit die KI sie behaelt
- **Iterieren**: Wenn das Ergebnis nicht passt — Ton, Laenge oder Input anpassen und erneut transformieren
- **Temperature**: Niedrig (0.0-0.3) fuer praezise Texte, hoeher (0.5-0.8) fuer kreativere Ergebnisse
- **Sprache**: Die Zielsprache in Schritt 3 setzen — der Quelltext kann in jeder Sprache sein
